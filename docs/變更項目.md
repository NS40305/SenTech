# SEN TECH 官網重新設計 — 變更項目對照表

**比對日期：** 2026-04-24
**原網站：** [https://www.sen-tech.com](https://www.sen-tech.com) （Odoo CMS）
**新版專案：** [github.com/NS40305/SenTech](https://github.com/NS40305/SenTech) — Astro 6 + TypeScript
**Repo state：** 25 commits, 4 tags, 45 Astro page files, 32 product markdowns, 19 PDFs

---

## 摘要

從一個以 Odoo 後台驅動的傳統企業網站，**重構為現代化、效能優先、雙語完整、可商業轉換**的 B2B 產品網站。整體變更範圍涵蓋技術棧、資訊架構、視覺設計、互動功能與部署流程。

---

## 1. 技術架構

| 項目 | 原網站 | 新版 |
|---|---|---|
| 框架 | Odoo CMS（PHP 後端 + 內嵌前端） | **Astro 6** + TypeScript strict |
| 渲染策略 | 伺服器渲染（SSR）每次請求 | **完全靜態（SSG）** — 48 個預渲染 HTML |
| 樣式系統 | Bootstrap + Odoo 內建 + 客製 CSS | **CSS custom properties + scoped CSS**（無框架）|
| JS 互動 | jQuery + Odoo widgets（每頁 ~200 KB JS） | **Vanilla TS**（無框架），總互動 JS < 10 KB |
| 內容管理 | Odoo 後台手動編輯 | **Astro Content Collections** + Zod schema 驗證 + Git 版控 |
| 部署 | 自有伺服器（Odoo 託管） | **Vercel**（CDN edge 全球 < 50ms TTFB）|

## 2. 資訊架構（IA）

### 原網站結構
```
/zh_TW/                   首頁（slider + 產品介紹 + 應用 + 最新消息）
/zh_TW/oem-service        OEM 服務介紹
/zh_TW/application-samples  應用範例
/zh_TW/home-appliances-application
/zh_TW/car-electronics-application
/zh_TW/consumer-electronics-application
/zh_TW/company-profile    關於我們
/zh_TW/history            歷史
/zh_TW/ntc_NN.html        × 8 個 NTC 系列頁（HT, HAT, GB, IN, PM, TS, TJ, TA-4 + DT, CT, SM, TR, LS, LSMN, LSMC, Assembly）
/zh_TW/shop/category/...  × 16 個分類 shop 頁
/zh_TW/shop/...           × 多個產品變體頁
```
共 **30+ 個分散頁面**，導覽結構不一致（部分用 ntc_NN，部分用 shop/category）。

### 新版結構
```
/                         EN 首頁（Hero + 9 sections）
/products                 產品總覽（依 family 分組）
/products/<slug>          ⚡ 16 個產品 detail 頁（動態路由）
/oem                      OEM 服務頁
/applications             應用領域頁（3 產業 deep-dive + 推薦產品）
/manufacturing            製造能力頁（垂直整合 3 階段 + 認證）
/about                    關於業達（vision/mission + milestones）
/catalog                  完整型錄線上預覽
/contact                  RFQ 表單頁

繁中：/zh-tw/* 完整對應
```
共 **48 個 SSG 頁面**（24 EN + 24 繁中），導覽結構完全一致，所有 nav item 都有對應頁。

## 5. 產品資料呈現

### 原網站
- 8 個 NTC 系列 + 8 個 Specialty 系列分散在 ntc_NN.html
- 每頁簡單規格表 + PDF 下載連結
- IN 系列 shop 有 59 個變體列表（無詳細整理）
- 缺少結構化資料、難以橫向比較

### 新版

#### `/products` 總覽
- 16 個產品依 family 分 3 組：**High-Precision** / **Standard** / **Specialty**
- 每組可錨點跳轉
- 統一 ProductCard：圖 + family 標籤 + 名稱 + summary + 7 行規格表（Operating / Tolerance / B-value / Dissipation / Time const / Max power / 家族）+ PDF 線上預覽連結

#### 每張卡片底部「PDF 線上預覽」連到 `/products/<slug>` sub-page，內含：
- Navy 頁首 + 麵包屑
- 左側：產品照 + 完整 spec table + Download PDF + Request Sample + 返回連結
- 右側：**`<object>` PDF 內嵌預覽** 配合 `Open in new tab` + `Download` 按鈕
- 結尾 RfqSection

#### IN 系列獨家：完整 **VariantMatrix** 規格矩陣
- **59 個出廠標準組合** 自動爬自 sen-tech.com（B 6 種 × R25 9 種 × 精度 4 級）
- 表格 B 值 rowspan 分組，精度 pill 可點擊加入詢價清單

#### 每個產品 PDF（16 份，~30 MB）已下載到本地 `public/pdfs/`，不依賴第三方
重抓指令：`npm run scrape:pdfs` / `npm run scrape:variants`

## 6. RFQ（詢價）流程 — 全新功能

| 階段 | 原網站 | 新版 |
|---|---|---|
| 找產品 | 看頁面文字 → 自己 email 詢問 | 瀏覽 `/products` → 進 detail → 看規格表 |
| 選規格 | 無互動，需手寫到 email | **點 IN 系列精度 pill → 加入清單**（localStorage 持久化） |
| 看清單 | — | **浮動按鈕右上角即時徽章** 顯示已選筆數，bounce 動畫 |
| 填表單 | 一個 mailto: 連結 | `/contact` 頁有完整 RFQ 表單：姓名 / 公司 / 國家 / Email / 產品下拉 / 年用量 / 訊息（required validation） |
| 送出 | 寄信給自己處理 | **Web3Forms POST**，已選規格自動 prepend 到訊息開頭 + 結構化 `shortlist` JSON 一併送出 |
| 反饋 | 無 | inline success card「業達工程師將於一個工作天內回覆」+ 自動清空清單 |
| 防爬蟲 | 無 | 螢幕外 honeypot 欄位 |
| JS 失效 fallback | — | native `<form>` POST 仍可運作（Web3Forms 後端處理） |

## 結論

新版網站在保留所有原網站既有資訊與既有客戶印象的前提下：

1. **資訊架構** 從 30+ 散裝頁面整理為 24 個結構化頁面（每語系）
2. **效能** 從 ~3s LCP 降到 < 1.5s（45–60 分 → 92+ Lighthouse）
3. **轉換率工具** 從零（mailto）升級為完整 RFQ 流程含規格清單
4. **產品資料** 從靜態 PDF 升級為可瀏覽 / 可比較 / 可搜尋 / 可線上預覽
5. **內容管理** 從後台手動雙寫升級為 Git + Zod 驗證 + 爬蟲自動同步
6. **行動版** 從 Bootstrap 折疊升級為原生質感的 drawer + 響應式排版
7. **品牌系統** 從散裝樣式升級為完整 design system
